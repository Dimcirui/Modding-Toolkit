# MHWI 一键生成 Chain — 设计方案

## 背景

MHWI 的物理链格式为 `.ctc`，基于 MHW Model Editor 插件操作，与 RE Engine 的 `.chain` 格式不同。本文档记录 MHWI 一键生成 Chain 功能的完整设计决策。

---

## 一、CTC 支链的已知缺陷（设计决策依据）

CTC 支链存在引擎层面的结构性缺陷，**不应自动生成支链**：

1. 模型加载时支链会丢失位置，从极远处飞回目标位置
2. 上述现象在过场动画切换分镜时造成严重卡顿，导致音画不同步
3. 支链表现不稳定，完全镜像、属性完全相同的支链可能有不同的物理表现

**结论：** 工具只生成线性链，遇到分叉链直接跳过并报告，由用户人工决定如何处理分叉。

---

## 二、一键生成 Chain 功能设计

### 新算子

`MHWI_OT_AutoCreateChains`，位于 `games/mhwi/operators.py`

**依赖检测：**
```python
hasattr(bpy.ops, 'mhw_ctc') and hasattr(bpy.ops.mhw_ctc, 'create_chain_from_bone')
```

**对话框参数：** 只选择目标 CTC Collection（无 settings 模式选项，因为每条 CTC chain 直接内嵌物理属性，无共享 settings 层级）

### 执行逻辑

```
1. 扫描当前集合，收集已存在 chain 绑定的链头骨骼名 → 幂等性集合
2. 找所有 chain_role = "head" / "branch_head" 的骨骼
3. 对每个链头：
   - 检查整条链是否存在分叉（任意骨骼有 >1 个物理子骨）
   - 有分叉 → 记入"跳过"列表，不生成
   - 无分叉 → 检查幂等性集合，已存在则跳过，否则调用
               mhw_ctc.create_chain_from_bone 生成
4. 汇报：成功创建 N 条，已存在跳过 N 条，因分叉跳过 N 条 + 列出骨骼名
```

### 幂等性设计

第二次运行时只补充之前因分叉跳过的链，不重复生成已有链。  
检查方式：扫描集合内所有 CTC chain 对象已绑定的链头骨骼名。

## 三、实验性物理工具（无需修改，MHWI 直接复用）

| 功能 | 状态 |
|---|---|

## 四、用户操作流程

```
实验性工具处理骨骼（颜色刷新、标记分叉等）
        ↓
运行一键生成 Chain
  → 线性链全部自动生成
  → 分叉链报告给用户（列出骨骼名）
        ↓
用户手动处理分叉（合并骨骼 / 删除支链 / 其他方案）
        ↓
再次运行一键生成 Chain（幂等，只补充之前跳过的链）
```

## 五、物理骨骼命名规范

### 部位与 ID 范围

| 部位 | 物理骨骼 ID 范围 | 实际可用数量 |
|---|---|---|
| helm | 无物理骨骼 | 0 |
| arm | MhBone_150~200 | ~50 |
| body | MhBone_150~245, 260~512（因骨骼总上限255，实际约150个） | ~150 |
| wst | MhBone_150~200 | ~50 |
| leg | MhBone_150~200 | ~50 |

**尾骨骼建议范围：** 150~245（与物理骨骼共享，但给物理骨腾出空间，建议物理骨在 150~200，尾骨在 201~245）

**body 建议范围：** 300~512（body 独有，用于区分其他部位）

**命名规则：** 只要 ID 符合范围且不重名即可，链内骨骼 ID **不要求**连续。

## 六、物理骨骼规范化功能（待设计）

### 功能目标

将所有物理骨骼自动重命名为符合规范的 `MhBone_xxx` 格式，并按部位分配 ID。

### 分配策略（已定）

- 默认以 body 骨架为起点（支持最多物理骨骼）
- 超出 body 限额时，将超出部分移动到其他部位的独立骨架（从原骨架复制基础骨架，不含物理骨）
- 超出单个部位限额（~50）时继续分配到下一个部位

### 分配原则（已定）

**以物理链为最小分组单位，不在链中间截断，不将同一链的骨骼拆分到不同部位。**

- 不应在遍历物理链到一半时，因超出数量直接截断到新部位
- 不应将同一条物理链的一部分留在原部位、另一部分移到新部位
- 应当以"整条链"为单位分组，同一功能区域的链尽量归属同一部位

### 未解决问题

**跨部位分组的决策逻辑：** 当总数超出 body 限制时，如何决定哪些链留在 body、哪些链移到其他部位？

- 按链遍历顺序贪心装箱？容易导致功能不相关的链被分在同一部位
- 按链的骨骼数量排序？
- 让用户手动指定分组？
