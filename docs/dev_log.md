# Dev Log

## MHWI 物理骨骼规范化分配算法

### 背景

MHWI 物理骨骼必须命名为 `MhBone_xxx` 格式，且 ID 必须符合各部位的范围限制。规范化功能需要将场景中命名任意的物理骨骼重命名并分配到合法的部位骨架中。

最核心的约束是：**物理链不能被拆分到不同部位**，整条链必须作为一个整体分配。

**总体使用流程：** 标记（骨骼颜色/chain_role）→ **规范化** → chain 生成

---

### 部位容量与 ID 规范

| 部位 | 物理骨 ID 范围 | 物理骨上限 | 备注 |
|------|--------------|----------|------|
| helm | 无 | 0 | |
| arm  | 150–200 | 50 | |
| body | 300–512 | ~150 | 骨骼总数须 ≤255；建议用 300–512 与其他部位区分 |
| wst  | 150–200 | 50 | |
| leg  | 150–200 | 50 | |

**尾骨骼：** 可命名在 150–245 范围内，但没有对应权重，物理效果不可见，惯例放在 201–245，为物理骨（150–200）腾出空间。

**槽位合法性判断：** 统计该部位所有物理骨数量，减去尾骨骼数量，结果 ≤50 为合法（body 特殊，不需要此计算，只需总骨骼数 ≤255）。

> 注意：`_is_mhwi_physics()` 目前检测 150–245 范围，这把物理骨和尾骨骼一并纳入了。规范化时需将两类区分开来单独统计。

---

### 区域检测

按解剖区域对物理骨骼分组，而非按链分组（链数量可能很多且每条都很短，按链分组 UI 极为臃肿）。

骨架天然分为四个区域：**头部、双臂、躯干、双腿**（arm_L/R 合并显示，leg_L/R 合并显示）。

**检测方法：** 对每个物理骨，沿父链向上遍历，找到最近的基础骨（preset bone），通过 `BoneMapManager.reverse_mapping` 映射回标准键，再查区域映射表。此逻辑与 `standard_ops.py` 中现有的父链遍历代码高度一致，可复用。

---

### 分配算法

#### 快速路径（绝大多数情况）

**当前骨架总骨骼数 ≤ 255** 时，直接全部分配到 body（使用 300–512 范围），无对话框，直接执行。

body 的真正瓶颈是骨骼总上限 255，而非物理骨的绝对数量，因此直接以此作为判断条件，无需单独统计物理骨数量。

#### 溢出路径

总骨骼数 > 255 时，展示区域分配 UI：

**默认分配规则：**
- 物理骨数量最多的区域 → body
- 其余区域按 arm → wst → leg 顺序依次分配
- 用户可通过下拉框自行调整每个区域的目标部位
- 任意部位超限时显示警告，确认按钮禁用，直到全部合法

**UI 示意（4 行以内）：**

```
+------------------------------------------+
| 物理骨骼分配                              |
|                                          |
| 区域     物理骨数   目标部位              |
| 头部        42    [body ▼]               |
| 双臂         8    [arm  ▼]               |
| 躯干        35    [wst  ▼]               |
| 双腿         0    (隐藏)                 |
|                                          |
| 容量:  body: 42/150  arm: 8/50           |
|        wst: 35/50   leg: 0/50            |
|                                          |
| [规范化]                                 |
+------------------------------------------+
```

不实现细分逻辑——如果用户的骨骼数量超出所有部位的承载范围，就让超限警告驱动用户自行精简，这是用户应当在规范化之前完成的工作。

---

### Blender 实现思路

- 阶段一算子：检测区域、计算分配方案，存入 `Scene` 的 `CollectionProperty`
- 阶段二面板：读取 `CollectionProperty`，显示区域列表 + 每项 `EnumProperty` + 容量状态
- 阶段三算子：读取方案，执行骨骼重命名与 ID 分配

## 物理链路径分解算法（`_decompose_chains`）

**文件**：`games/mhws/operators.py`

### 背景

RE Chain Editor 的 `chain_from_bone()` 默认模式只接受单个链头骨骼作为输入，从链头沿子骨方向自动向下遍历生成一条链。当物理骨骼结构存在分叉时（一根骨骼有多个物理子骨），单次调用无法处理多条路径，需要借助实验模式（`experimentalPoseModeOptions = True`）并手动选中骨骼集合后调用 `chain_from_bone()`。

`_decompose_chains` 的目的是将任意拓扑的物理骨骼树自动分解为若干条线性路径，每条路径对应一次 `chain_from_bone()` 调用。

---

### 输入 / 输出

```
输入：
  head_pb       — 链首 PoseBone（chain_role == "head" 或 "branch_head"）
  armature      — 骨架对象
  physics_bones — 物理骨骼名称集合（= 骨架全部骨骼 - 预设骨骼）

输出：
  list of list[str]
  每个子列表是一条从某个链首到末端（含 _End 骨骼）的骨骼名有序列表
```

---

### 核心规则

| 情况 | 处理 |
|------|------|
| 当前骨骼无物理子骨（叶骨） | 路径在此结束；若存在同名 `_End` 骨骼则追加到末尾 |
| 当前骨骼有 1 个物理子骨（线性链） | 直接继续，子骨加入当前路径 |
| 当前骨骼有 ≥2 个物理子骨（分叉），且有子骨标记 `main_continue` | `main_continue` 子骨继续当前路径；其余子骨各自从**空路径**开始递归 |
| 当前骨骼有 ≥2 个物理子骨（分叉），无 `main_continue` 标记 | 当前骨骼为主链终点（追加 `_End`）；所有子骨从空路径开始递归 |

> **关键细节**：`_End` 骨骼（名称以 `_End` 结尾）虽然在 `physics_bones` 集合内，但必须从 `physics_children` 中排除，否则每根骨骼都会被检测到有子骨，永远走不到叶骨逻辑，导致所有骨骼单独成链。`_End` 只在叶骨路径结束时作为终结符追加。

---

### 递归示意

```
A → B → C → D        (线性链，无分叉)
→ 输出：[[A, B, C, D, D_End]]

A → B → C → D        (B 有两个子骨，C 标记为 main_continue)
         ↘ E
→ walk(A, [])
    walk(B, [A])
      walk(C, [A, B])        ← main_continue
        walk(D, [A, B, C])
          叶骨 → 追加 D_End → 路径完成 [A, B, C, D, D_End]
      walk(E, [])             ← 支链，空路径开始
        叶骨 → 追加 E_End → 路径完成 [E, E_End]
→ 输出：[[A, B, C, D, D_End], [E, E_End]]

A → B → C            (B 有两个子骨，无 main_continue)
         ↘ D
→ walk(A, [])
    walk(B, [A])
      分叉无标记 → B 是终点，追加 B_End → 路径完成 [A, B, B_End]
      walk(C, [])   walk(D, [])   ← 所有子骨从空路径开始
→ 输出：[[A, B, B_End], [C, C_End], [D, D_End]]
```

---

### 与 `chain_role` 的配合

- `head` / `branch_head`：由 `_detect_chain_roles()` 自动检测写入（`core/standard_ops.py`），作为 `_decompose_chains` 的入口骨骼
- `main_continue`：用户手动标记，决定分叉处主链方向
- 普通体骨（无属性）：只参与路径拼接，无需标记

---

### 调用点

`MHWS_OT_AutoCreateChains.execute()`（`games/mhws/operators.py`）：

1. 收集所有 `chain_role in ("head", "branch_head")` 的骨骼作为链首
2. 对每个链首调用 `_decompose_chains` 得到路径列表
3. 路径数 == 1 → 普通模式（只选链头，`experimentalPoseModeOptions = False`）
4. 路径数 > 1 → 实验模式（选中路径上所有骨骼，`experimentalPoseModeOptions = True`）
