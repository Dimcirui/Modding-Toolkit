## 概述

Modding Toolkit 是一款面向游戏模组（Mod）制作的综合型 Blender 插件，旨在解决跨游戏、跨骨架类型的模型移植与批量导出中的各类技术问题。

### 核心能力

| 系统 | 能力 |
|------|------|
| **通用骨架转换器** | 通过 X→Y 双预设架构，将任意来源模型的骨架/顶点组转换为任意目标游戏格式 |
| **姿态转换系统** | 独立的姿态变换工具集，支持简易方向修正、RE Engine 矩阵归零、通用姿态录制与复用 |
| **可视化预设编辑器** | 在 GUI 中直接创建和编辑骨骼映射预设，无需编写代码 |
| **游戏专用模块** | 针对 MHWI/MHWs/RE4/RE9 的一键批量导出、MDF2 贴图处理、Chain 创建、Face 处理等 |

### 支持的游戏

- 怪物猎人世界：冰原 (MHWI)
- 怪物猎人：荒野 (MHWs)
- 生化危机4重制版 (RE4R)
- 生化危机：镇魂曲 (RE9/RE:Requiem)
- 街霸6 (SF6) — 通过预设支持
- 鬼泣5 (DMC5) — 通过预设支持
- 绝地潜兵2 (HD2) — 通过预设支持
- 怪物猎人崛起：曙光 (MHRS) — 预设支持

### 支持的来源模型类型

通过 JSON 预设，支持导入以下类型的模型：

- **VRChat** 标准骨架
- **MMD**（MikuMikuDance）骨架
- **明日方舟：终末地** 骨架
- **街霸6** 源骨架
- 以及任何可通过自定义预设描述的自定义骨架

## 安装与依赖

### 安装步骤

1. 从 [Releases](https://github.com/SheeranW/MHW-Modding-Toolkit/releases) 页面下载最新的 **ZIP** 包
2. 打开 Blender，进入 `编辑` → `偏好设置` → `插件`
3. 点击右上角 `安装` 按钮，选择下载的 ZIP 文件
4. 在插件列表中搜索 "Modding Toolkit"，勾选启用

### 依赖插件

以下部分功能需要额外安装配套插件才能使用：

| 依赖插件 | 用途 |
|----------|------|
| **RE Mesh Editor** | MHWs/RE4/RE9 的批量导出、MDF2 处理 |
| **MHW Model Editor** | MHWI 的批量导入导出、Chain 生成、物理骨骼处理 |
| **RE Chain Editor** | MHWs/RE4/RE9 的 RE Chain 一键创建 |

缺少对应插件时，UI 中的相关按钮会呈灰色不可用状态，并有提示信息。

## 基础工具

基础工具区提供不区分游戏类型的通用骨骼/网格操作。

### 扭转归零 (Roll=0)

**操作**：在编辑模式下选中骨骼，点击「扭转归零」
**效果**：递归将选中骨骼及其所有子孙骨骼的 Roll 值设为 0

### 添加尾骨

**操作**：在编辑模式下选中骨骼，点击「添加尾骨」
**效果**：在选中骨骼的末端沿 Z+ 方向添加一根垂直小骨骼（长度 0.05）

### 镜像对齐 X

**操作**：在编辑模式或姿态模式下**恰好选中 2 根骨骼**，点击「镜像对齐 X」
**效果**：以 X+ 侧骨骼为基准，将 X- 侧骨骼的 head/tail/roll 进行镜像对齐

### 骨链简化

**操作**：在编辑模式下选中一串链上的骨骼，点击「骨链简化」
**效果**：
1. 将选中骨骼按链接关系自动分组为链
2. 按「两两配对」进行合并删除（保留前一根，将后一根的权重并入前一根）
3. 自动跳过"尾骨"（没有任何顶点权重的骨骼，通常为功能性轴骨）
4. 例如选中 5 根骨骼 → 合并为 3 根

### 合并到激活骨

**操作**：在编辑模式下选中多根骨骼，**最后点击的那根为保留目标**（激活骨骼），点击「合并到激活骨」
**效果**：将其余所有选中骨骼的顶点权重合并到激活骨骼，然后删除其余骨骼

### 合并链到激活链

**操作**：在编辑模式下选中多条链的链首骨骼，确保其中一根为激活骨骼，点击「合并链到激活链」
**效果**：按逐骨位置顺序将其他链的骨骼权重合并到激活骨骼所在的链，长度超出的骨骼合并到链末

### 骨架对齐 (三种模式)

**前提**：同时选中两个骨架（后选的、黄色高亮显示的为目标）

| 模式 | 效果 |
|------|------|
| **位置** | 仅将目标骨骼的头部（head）移动到源骨骼对应位置，不改方向 |
| **位置+扭转** | 对齐头部位置和 Roll 值 |
| **完全** | 完全对齐 head、tail、roll，即骨骼完全重合 |

> 对齐匹配基于**骨骼名称**——名称相同的骨骼会被对齐。

## 通用骨架转换系统

这是本插件的核心系统，采用 **X → Y 双预设架构**，将任意来源（X）模型的骨架和顶点组转换到任意目标游戏（Y）的格式。

### 核心概念

```
来源模型 (X)              目标游戏 (Y)
┌──────────┐             ┌──────────┐
│ VRChat   │             │ Monster  │
│ 骨架     │  ────映射───> │ Hunter   │
│ 顶点组   │             │ Wilds     │
└──────────┘             └──────────┘
```

- **X 预设**：描述来源模型的骨架结构，定义每根骨骼在标准名下的来源名
- **Y 预设**：描述目标游戏的骨架结构，定义每根骨骼在标准名下的目标名
- **模糊匹配**：骨骼名中的 `_`、`.`、空格会被归一化，因此 `UpperArm.L` 和 `UpperArm_L` 视为等价

### 使用 X → Y 转换

#### 第 1 步：选择预设

在「通用标准转换」区域中：
1. **来源预设 (X)**：从下拉菜单选择你的来源模型类型（如 VRChat、MMD、终末地等）
2. **目标游戏 (Y)**：选择目标游戏（如 怪猎世界、怪猎荒野、生化4 等）

> 预设下拉菜单支持分类分组，大型预设库可通过分类快速筛选。

#### 第 2 步：对齐骨骼 (Snap)

**操作**：
1. 先选中来源（X）骨架
2. 按住 `Ctrl` 加选目标（Y）骨架（目标骨架应为黄色高亮/激活状态）
3. 点击「对齐骨骼 [X+Y, 双骨架]」

**效果**：将目标骨架的身体骨骼刚性对齐到来源骨架的对应位置。父级移动后子级会跟随移动，确保整体对齐准确。

#### 第 3 步：重命名顶点组 (Direct Convert)

**操作**：
1. 选中需要处理的网格对象（可多选）
2. 确保 X 和 Y 预设已正确选择
3. 点击「重命名顶点组 [X+Y]」

**效果**：
1. 将网格上来源骨骼（X）的顶点组直接重命名为目标游戏（Y）的名称
2. 自动合并"辅助骨骼"（aux）的权重到主骨骼
3. 模糊匹配骨骼名，兼容不同命名习惯

> **提示**：这两个核心步骤通常就可以完成基本的模型转换。下方「实验性功能」和「游戏专用工具」用于更进阶的场景。

### 实验性功能

点击「实验性功能」折叠区域展开更多高级操作：

#### 骨架清理

| 功能 | 说明 |
|------|------|
| **物理权重降级 [X]** | 将物理骨骼的顶点组权重沿父级链向上合并到最近的基础骨骼。用于不需要物理效果或目标游戏不支持的场景 |
| **剔除非基础骨骼 [X]** | 删除骨架中所有不在 X 预设中的骨骼（即删除物理骨骼） |
| **基础骨骼改名 [X+Y]** | 将骨架上的基础骨骼从来源名直接重命名为目标游戏名。用于手动对齐工作流 |

#### 物理链工具

| 功能 | 说明 |
|------|------|
| **移植物理骨骼 [X+Y, 双骨架]** | 将来源骨架的物理骨骼复制到目标骨架。自动生成 `_End` 末端骨，统一竖直化（Z+），并智能重建父子关系 |
| **合并到父骨** | 将选中骨骼的权重合并到其父骨骼，并删除自身。用于清理功能性根骨 |
| **标记为主链延伸** | 在分叉处标记哪根子骨是主链方向，未标记的被视为支链头。染色为琥珀金色 |
| **清除标记** | 清除骨骼的 chain_role 标记，恢复为普通体骨（深蓝色） |
| **刷新骨骼颜色 [X]** | 根据骨骼拓扑和 chain_role 自动染色：链首=天蓝，支链头=紫色，主链延伸=琥珀色，普通体骨=深蓝 |

#### 骨骼显示控制 [X]

三态切换选中骨架的骨骼可见性：
- **全显**：显示所有骨骼
- **仅基础骨**：隐藏物理骨，只显示预设中的基础骨骼
- **仅物理骨**：隐藏基础骨，只显示物理骨骼

#### 映射详情预览

展开「显示映射细节」后，当前选中的骨架会显示 X 和 Y 预设的详细映射预览。按「躯干和头部/手臂/腿部/手指」分组展示，绿色 ✓ 表示匹配成功，红色 ✗ 表示缺失。

## 姿态转换系统

独立于骨架转换的姿态变换工具集，分为三层。

### 第一层：方向计算

**用途**：简易工具，用于将 MMD 等 A-Pose 模型的上臂旋转到水平方向（A-Pose → T-Pose）

**操作**：
1. 选择骨架预设（用于识别上臂骨骼名）
2. 点击「方向计算」

> 如果此工具无法正确工作，请使用下方的「姿态变换记录器」通用方案。

### 第二层：RE Engine 矩阵归零

**用途**：适用于 RE Engine 系列游戏（荒野/街霸6/生化4），将肢体骨骼的旋转矩阵重置为单位矩阵。

**操作**：选择骨架预设后点击「RE Engine 矩阵归零 (生化9除外)」

> 注意：此功能不适用于生化危机9（生化9使用不同的骨骼矩阵方案）。

### 第三层：姿态变换记录器

核心姿态转换工具，采用「录制一次，永久复用」设计。

#### 录制变换

**操作**：
1. 准备两个同类型但不同姿态的骨架（如 A-Pose 和 T-Pose）
2. **先选 A 姿态骨架，再 Ctrl 加选 B 姿态骨架**（B 为活动对象）
3. 点击「录制变换 (选两个骨架)」
4. 在弹出的对话框中输入记录名称（如 "MMD A-Pose到T-Pose"）
5. 点击确定

**原理**：计算每根骨骼相对于父级的局部旋转差（delta），仅保存变化量超过阈值的骨骼。

**存储位置**：`assets/presets/pose/` 目录下，插件更新不会删除已有记录。

#### 应用变换

录制完成后，选择姿态记录文件后可用：

| 方向 | 效果 |
|------|------|
| **▶ 正向 (A→B)** | 将当前骨架从 A 姿态转换为 B 姿态 |
| **逆向 (B→A)** | 将当前骨架从 B 姿态转换为 A 姿态 |

**操作**：选中一个骨架（无需是录制时的那个，同类型即可），从下拉选择记录文件，点击正向或逆向应用。

> 变换通过「骨架预设」建立骨骼名桥接：录制时的骨骼名 → 标准键 → 目标骨架的骨骼名，因此不同命名的同类型骨架也可复用。

## 可视化预设编辑器

在 MOD Toolkit 侧栏的第二个标签页中，提供在 GUI 中直接创建和编辑骨骼映射预设的能力。

### 布局说明

```
┌────────────────────────────────────┐
│ 管理现有预设 (Manage)               │
│  [X] [Y]  模式切换                 │
│  [预设下拉] [读取/编辑] [删除]      │
│  [打开预设文件夹] [复制为 X/Y]       │
├────────────────────────────────────┤
│ 编辑器工作区                        │
│  [保存名输入框]  [保存]            │
│  [清空并初始化列表]                 │
│  [搜索过滤]  [L -> R 镜像]          │
├────────────────────────────────────┤
│ 骨骼映射列表（按分组展示）           │
│  躯干和头部:                       │
│    盆骨: [Hips]  ✗ ↕ Aux(0) +     │
│    脊椎: [Spine]  ✗ ↕ Aux(0) +    │
│    ...                             │
│  手臂、腿部、手指...               │
└────────────────────────────────────┘
```

### 编辑模式 (X/Y)

- **X 模式**：编辑来源预设（存储在 `assets/presets/import/`）
- **Y 模式**：编辑目标预设（存储在 `assets/presets/bone/`）

### 基本操作

1. **创建新预设**：
   - 点击「清空并初始化列表」为全部 58 个标准骨骼生成空槽位
   - 输入保存名称
   - 逐槽位点击吸管图标（👁）从 3D 视口中选取骨骼
   - 可点击 `+` 按钮批量添加辅助骨骼（权重会被合并的骨骼）
   - 点击「L -> R」自动从左侧映射镜像生成右侧映射

2. **编辑现有多余**：
   - 选择 X/Y 模式，在下拉菜单中选择预设
   - 点击「读取/编辑」加载到编辑器

3. **管理预设**：
   - 「打开预设文件夹」：在系统文件管理器中打开预设目录
   - 「转换」：将 X 预设复制为 Y 预设（或反之），保存到对应的另一个目录

### 辅助骨骼 (Auxiliary Bones)

辅助骨骼是其权重会被合并到主骨骼的附加骨骼。例如 VRChat 模型中的 `LeftArmTwist` 可以设为 `upperarm_L` 的辅助骨，转换时其顶点权重会自动并入上臂。

## MHWI 专用工具

怪物猎人世界：冰原（Monster Hunter World: Iceborne）的专用功能。

> **依赖**: 需要安装 **MHW Model Editor** 插件

### 对齐非物理骨骼

选中两个骨架后使用，跳过 MHWI 物理骨骼（ID 150-245），只对齐基础身体骨骼。

### 物理骨骼规范化

| 功能 | 说明 |
|------|------|
| **拆分物理骨** | 按解剖学区域（头发/胸/腰/裙等）将物理骨骼拆分到独立的子骨架中，解决 MHWI 单骨架 255 骨骼的上限问题 |
| **一键重命名** | 将物理骨骼统一重命名为 `MhBone_xxx` 格式（body 骨骼用 150-245 段，非 body 用 246+ 段） |

### 一键创建 Chain

自动从带有 chain_role 标记的物理骨骼创建 MHW Model Editor 格式的 CTC Chain 数据。

### 批量导出装备

**操作方法**：
1. 展开 MHWI Tools 区
2. 选择「装备包」（JSON 文件，定义装备列表）和「位阶」（上下位/大师位/整套幻化）
3. 选择「性别」（女/男/双性）
4. 从列表中选择要导出的装备
5. 点击「批量导出装备」打开导出对话框

**导出内容**：每个装备可导出 5 个部位（手臂/腿/身体/头盔/腰）的 mod3（模型）、mrl3（材质）、ctc/ccl（物理）文件。

### 批量导入装备

从游戏目录中批量导入 MHWI 装备到 Blender，操作流程与导出对称。

### MRL3 处理/生成器

- **MRL3 处理器**：批量更新 MRL3 材质集合中的贴图绑定路径并转换贴图格式
- **MRL3 生成器**：从 Blender 材质节点生成 MRL3 材质文件

## MHWilds 专用工具

怪物猎人：荒野（Monster Hunter Wilds）的专用功能。

> **依赖**: 需要安装 **RE Mesh Editor** 和 **RE Chain Editor** 插件

### Endfield 面部改名

将明日方舟：终末地（Endfield）格式的 118 个面部顶点组名称一键转换为 MHWilds 格式。

### 面部权重简化

将细分面部骨骼的顶点权重按比例合并到主控制骨骼，简化面部骨骼结构。

### MHWs Batch Exporter

**操作方法**：
1. 展开 MHWilds Tools 区
2. 选择「装备包」（`assets/mhws/armor_sets/` 下的 JSON 文件）
3. 选择「套装种类」（男猎男套/男猎女套/女猎男套/女猎女套）
4. 选择「装备」
5. 点击「MHWs Batch Exporter」打开导出对话框

**导出内容**：每个装备 5 个部位（手臂/身体/头盔/腿/腰）的 4 种文件：

| 文件类型 | 扩展名 | 说明 |
|----------|--------|------|
| **mesh** | `.mesh.241111606` | 模型文件 |
| **MDF2** | `.mdf2.45` | 材质文件 |
| **chain2** | `.chain2.14` | 物理链文件 |
| **clsp** | `.clsp.3` | 碰撞体文件 |

### BoneSystem 集成

在批量导出面板中可启用「使用 Bonesystem」，配置：
- **FBXSkel 定义名**：写入 JSON 的 `FbxPath` 字段和 `.fbxskel.7` 文件名
- **骨架**：选择用于生成的 MHWs 角色骨架
- **隐藏/绑定选项**：控制面部、头发、投射器的隐藏和绑定

导出时会同时生成：
- `.fbxskel.7` 文件
- `reframework/data/BoneSystem/{id}.json` 配置文件

### 未选项使用空模型

启用后，导出时未绑定集合的部位会自动复制内置的空文件（blank mesh/MDF2/chain2/clsp）到目标路径，而非跳过。

### 一键创建 RE Chain

自动检测标记为链首的物理骨骼，弹出集合选择器，通过 RE Chain Editor 为每条物理链创建 Chain Settings + Chain Group。

### MDF2 处理/生成器

参见下方 [MDF2 贴图处理器通用说明](#mdf2-贴图处理器通用说明)。

## RE4 专用工具

生化危机4重制版（Resident Evil 4 Remake）的专用功能。

> **依赖**: 需要安装 **RE Mesh Editor** 和 **RE Chain Editor** 插件

### 假头法 (FakeBone)

RE4 的特殊骨骼生成方法，通过约束法从原生 fbxskel 骨架数据自动生成身体和手指的 End 骨骼。

**操作**：展开 RE4 Tools → 「假头法」区域，点击「生成假骨骼」

### RE4 Batch Exporter

**操作方法**：
1. 选择「Export Scheme」（`assets/export_schemes/re4/` 下的 JSON 方案文件）
2. 配置「使用假头法」和「未选项使用空模型」选项
3. 点击「RE4 Batch Exporter」打开导出对话框

导出支持 mesh/MDF2/SFUR/chain2/clsp 五种文件类型。

### 一键创建 RE Chain

与 MHWs 相同，通过 RE Chain Editor 自动创建物理链。

### MDF2 处理/生成器

参见下方 [MDF2 贴图处理器通用说明](#mdf2-贴图处理器通用说明)。

## RE9 专用工具

生化危机：镇魂曲（Resident Evil: Requiem，代号 RE9）的专用功能。

> **依赖**: 需要安装 **RE Mesh Editor** 和 **RE Chain Editor** 插件

### 同步子级朝向及扭转

选中任意骨骼后点击，将其所有子骨骼的朝向和 Roll 同步对齐到父骨骼方向。用于修正子骨骼的扭转朝向。

### RE9 Batch Exporter

- 通过 JSON 方案批量导出 mesh/MDF2/SFUR/chain2/clsp 文件
- 支持「简化模式」：按组绑定集合，无需逐条目配置
- 支持「未选项使用空模型」：unbound slots 自动复制内置空文件
- 简化模式下的 "empty" 类型条目直接使用内置空文件

### 一键创建 RE Chain

与 MHWs、RE4 相同。

### MDF2 处理/生成器

参见下方 [MDF2 贴图处理器通用说明](#mdf2-贴图处理器通用说明)。

## MDF2 贴图处理器通用说明

MDF2 处理器和生成器在 MHWs/RE4/RE9 中有各自的版本，但共享相同的使用逻辑。以下以通用视角说明。

### MDF2 处理器 (Tex Processor)

**用途**：批量更新 MDF2 材质集合中的贴图绑定路径，并将来源图像一步转换为游戏可用的 `.tex` 文件。

#### PBR 输入配置

每个材质可独立配置以下 PBR 输入：

| 输入 | 说明 | 颜色空间 |
|------|------|----------|
| **基础色 (Albedo)** | 固有色贴图 | sRGB |
| **Alpha 遮罩** | 透明度贴图 | - |
| **法线 (Normal)** | 法线贴图（含 GL→DX 翻转开关） | Linear |
| **粗糙度 (Roughness)** | 粗糙度/平滑度贴图 | Linear |
| **金属度 (Metallic)** | 金属度贴图 | Linear |
| **AO** | 环境光遮蔽贴图 | Linear |
| **自发光 (Emissive)** | 自发光贴图 | sRGB |

对于 Alpha、粗糙度、金属度、AO 类单通道输入：
- 可选择来源通道（R/G/B/A）
- 可开启反相（例如平滑度→粗糙度、绝缘度→金属度）

#### 贴图槽位模式

每个材质中的每个贴图槽位（如 AlbedoMap、NormalMap 等）有 4 种处理模式：

| 模式 | 行为 |
|------|------|
| **PBR转换 (COMPOSE)** | 从 PBR 输入自动合成通道布局（R/G/B/A 各自来源），转换为 `.tex` |
| **直接选择 (DIRECT)** | 直接选择任意图片/DDS/TEX 作为输入 |
| **默认空贴图 (DEFAULT)** | 写入游戏内空贴图路径（自动检测） |
| **不修改 (SKIP)** | 保持 MDF2 中的现有路径不变 |

#### 其他特性

- **法线翻转 (GL→DX)**：按材质独立开关，自动翻转法线 G 通道
- **刷新自动检测**：刷新时自动识别现有空贴图路径并设为 DEFAULT
- **复制/粘贴**：支持材质配置的复制粘贴
- **状态持久化**：切换 MDF2 集合时各自独立保留配置
- **输出格式**：颜色/自发光槽 → BC7_UNORM_SRGB，线性槽 → BC7_UNORM

### MDF2 生成器 (Generator)

**用途**：从 Blender 的 Principled BSDF 材质节点自动生成 MDF2 材质集合和对应的贴图。

**操作**：
1. 选择 MDF2 集合
2. 选择导出目标（Natives 根目录）
3. 点击生成

**PBR 检测策略**：
1. **DIRECT**：直接从 Principled BSDF 节点的图像纹理输入
2. **SOLID**：节点未连接图像时使用固定值（如白色 1.0）
3. **BAKE**：通过 Cycles 渲染烘焙材质结果到贴图

## 常见工作流程

### 工作流一：VRChat 模型 → MHWilds

```
1. 将 VRChat 模型导入 Blender
2. 来源预设选 "VRChat"，目标预设选 "怪猎荒野"
3. 导入并调整 MHWs 目标骨架
4. 选中 VRChat 骨架 → Ctrl 选中 MHWs 骨架
5. 点击「对齐骨骼」→ 目标骨架对齐到来源
6. 选中所有网格 → 点击「重命名顶点组」
7. （可选）展开实验性功能：
   - 物理权重降级（如不需要物理）
   - 剔除非基础骨骼
   - 移植物理骨骼（如需要保留物理）
8. 使用「基础骨骼改名」统一骨骼名称
```

### 工作流二：MMD A-Pose → T-Pose → RE4

```
1. 导入 MMD 模型（A-Pose）
2. 姿态转换区：设置骨架预设为 MMD
3. 点击「方向计算」旋转上臂到水平
4. 点击「RE Engine 矩阵归零」重置肢体矩阵
5. 通用转换区：来源预设 "MMD"，目标预设 "生化4"
6. 对齐 → 重命名顶点组
7. RE4 工具区：如有需要，点击「生成假骨骼」
8. 使用 RE4 Batch Exporter 批量导出
```

### 工作流三：姿态录制与批量应用

```
1. 准备两个同类型骨架（A-Pose 和 T-Pose）
2. 姿态转换区 → 点击「录制变换」
3. 先选 A 骨架，Ctrl 加选 B 骨架
4. 在弹出的对话框中输入名称 "MyModel_A_to_T"
5. 点击确定保存
6. 之后任何同类型骨架：
   - 从姿态记录下拉选择 "MyModel_A_to_T"
   - 点击「正向」即转为 T-Pose
   - 点击「逆向」即转回 A-Pose
```

### 工作流四：MHWI 装备批量导出

```
1. 确保安装了 MHW Model Editor
2. 在场景中为每个部位/文件类型绑定对应的集合
3. 展开 MHWI Tools → 选择装备包 → 位阶 → 性别 → 装备
4. 点击「批量导出装备」打开对话框
5. 按部位和文件类型检查绑定是否正确
6. 点击导出
```

### 工作流五：创建自定义预设

```
1. 打开「预设编辑器」标签页
2. 选择 X 或 Y 编辑模式
3. 点击「清空并初始化列表」
4. 在 3D 视口中选中来源/目标骨架
5. 对每个标准骨骼槽位，点击吸管图标选取对应骨骼
6. 需要合并的辅助骨骼点 + 按钮添加
7. 左侧填完后点击「L -> R」自动镜像
8. 输入保存名称，点击「保存」
```

## 预设文件格式

### 骨骼映射预设 (X/Y)

位置：
- X 预设：`assets/presets/import/*.json`
- Y 预设：`assets/presets/bone/*.json`

```json
{
  "preset_info": {
    "name": "VRChat",
    "type": "X_PRESET",
    "version": "2.0"
  },
  "mappings": {
    "pelvis": {
      "main": ["Hips"],
      "aux": []
    },
    "upperarm_L": {
      "main": ["LeftUpperArm", "UpperArm_L"],
      "aux": ["LeftArmTwist"]
    },
    "upperarm_R": {
      "main": ["RightUpperArm", "UpperArm_R"],
      "aux": ["RightArmTwist"]
    }
  },
  "exclude": ["WeaponRoot", "CameraRoot"]
}
```

| 字段 | 说明 |
|------|------|
| `preset_info` | 预设元数据：名称、类型、版本 |
| `mappings` | 标准骨骼名 → 来源/目标骨骼配置 |
| `mappings.{key}.main` | 主要骨骼的候选名列表（按优先级排列，第一个匹配成功） |
| `mappings.{key}.aux` | 辅助骨骼列表（其权重会被合并到主骨） |
| `exclude` | 排除的骨骼列表（非基础骨也非物理骨，如功能根骨） |

### 姿态变换记录

位置：`assets/presets/pose/*.json`

```json
{
  "type": "pose_relative_transform",
  "version": "2.0",
  "source_a": "A骨架名称",
  "source_b": "B骨架名称",
  "description": "A-Pose -> T-Pose",
  "bone_count": 24,
  "transforms": {
    "Shoulder_L": [0.923, 0.0, 0.0, 0.383],
    "UpperArm_L": [0.707, 0.0, 0.0, 0.707]
  }
}
```

| 字段 | 说明 |
|------|------|
| `type` | 固定为 `pose_relative_transform` |
| `transforms` | `{骨骼名: [w, x, y, z]}` 四元数格式的旋转差 |
| `bone_count` | 有效变化的骨骼数量（去除接近单位四元数的） |

### 装备包 (Armor Sets)

位置：
- MHWI：`assets/mhwi/armor_sets/*.json`
- MHWs：`assets/mhws/armor_sets/*.json`

MHWs 示例：
```json
{
  "armor_sets": [
    {
      "id": "armor_001",
      "name": "某套装",
      "variants": {
        "ff": {
          "armor_id": "em001_000",
          "base_path": "natives/STM/...",
          "parts_mask": []
        }
      }
    }
  ]
}
```

### 导出方案 (Export Schemes)

位置：
- RE4：`assets/export_schemes/re4/*.json`
- RE9：`assets/export_schemes/re9/*.json`

## 更新机制

本插件内置了基于 [Blender Addon Updater](https://github.com/CGCookie/blender-addon-updater) 的自动更新系统。

### 自动检查

- 在 Blender 偏好设置中可配置自动检查更新
- 有新版本时会在面板顶部显示通知

### 手动检查

（如有配置，可通过面板底部的更新按钮手动检查）

### 预设文件的安全性

- 所有用户创建的预设（`assets/presets/` 目录下）存储在插件目录内部
- **插件更新不会覆盖或删除**用户创建的所有 JSON 预设文件
- 更新后新版本预设会自动添加到预设库中

## 58 个标准骨骼参考

插件内部定义的标准骨骼名，覆盖常规人体骨骼的全部范围：

| 分组 | 标准骨骼名 |
|------|-----------|
| **躯干** | pelvis, spine_01, spine_02, spine_03, neck, head |
| **上肢 (L/R)** | clavicle, upperarm, forearm, hand |
| **下肢 (L/R)** | thigh, shin, foot, toe |
| **手指 (L/R)** | thumb_01~03, index_01~03, middle_01~03, ring_01~03, pinky_01~03 |

> 注：spine_03 为可选骨骼，某些目标游戏（如 MHWI/MHWs）不支持时，转换过程中将自动降级到 spine_02。
